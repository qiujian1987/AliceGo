# 代码审查

## 1. 收集变更信息

### 获取变更文件
```bash
git diff --name-only
git diff --stat
```

### 查看详细diff
```bash
git diff [文件路径]
```

## 2. 审查维度

### 逻辑正确性
- [ ] 业务逻辑是否正确
- [ ] 边界条件是否处理
- [ ] 异常情况是否考虑
- [ ] 并发问题是否存在

### 代码质量
- [ ] 命名是否清晰
- [ ] 函数是否过长
- [ ] 是否有重复代码
- [ ] 注释是否必要

### 安全风险
- [ ] 输入是否验证
- [ ] SQL注入风险
- [ ] XSS风险
- [ ] 敏感信息泄露

### 性能问题
- [ ] 是否有不必要循环
- [ ] 是否滥用全局状态
- [ ] 资源是否及时释放
- [ ] 是否有内存泄漏

### 测试覆盖
- [ ] 核心逻辑是否有测试
- [ ] 边界条件是否有测试
- [ ] 错误处理是否有测试

## 3. 按文件输出审查结果

```
## [文件名]

### 潜在问题
- 行号：问题描述

### 安全风险
- 行号：风险描述

### 优化建议
- 行号：建议内容
```

## 4. 总结

### 高优先级（必须修复）
### 中优先级（建议修复）
### 低优先级（可选优化）

### 总体评价
- 代码质量：优秀/良好/一般/较差
- 主要问题：xxx
- 建议：xxx
