# 工程标准

## 1. 代码规范

### 命名规范
- 文件命名：kebab-case
- 组件/类命名：PascalCase
- 变量命名：camelCase
- 常量命名：UPPER_SNAKE_CASE

### 长度规范
- 函数长度：不超过 50 行
- 类/组件长度：不超过 300 行
- 文件长度：不超过 500 行
- 嵌套深度：不超过 3 层

### 注释规范
- 公共 API：JSDoc 注释
- 复杂逻辑：解释性注释
- 文件头部：文件说明注释
- 更新代码时同步更新注释

## 2. 代码质量工具

### 前端
- **ESLint**：代码风格和语法检查
- **Prettier**：代码格式化
- **TypeScript**：类型检查
- **Stylelint**：CSS 风格检查

### 后端
- **ESLint**：代码风格和语法检查
- **Prettier**：代码格式化
- **TypeScript**：类型检查
- **Jest**：单元测试

## 3. 测试策略

### 测试类型
- **单元测试**：测试单个函数和组件
- **集成测试**：测试模块间的交互
- **端到端测试**：测试完整的用户流程

### 测试覆盖率目标
- 核心业务逻辑：≥ 80%
- 公共 API：100%
- 数据模型：100%

### 测试质量
- 测试独立，不相互依赖
- 测试名称描述清晰
- 使用真实数据，避免 mock 过度

## 4. 开发流程

1. 需求分析 → 架构设计 → 任务分配
2. 代码开发 → 代码审查 → 测试执行
3. 集成部署 → 监控维护

## 5. CI/CD 流程

1. 代码提交 → 质量检查 → 测试执行
2. 构建验证 → 部署准备 → 环境检查
3. 部署执行 → 健康检查 → 回滚机制

## 6. 环境配置

- **开发环境**：local
- **测试环境**：staging
- **生产环境**：production
