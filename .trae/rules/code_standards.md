# 代码标准

## 1. 准确性标准

### 语法正确性
- 生成代码必须通过语言编译/解释器检查
- TypeScript/JavaScript 通过 ESLint 检查
- Python 通过 PEP8 和 pylint 检查
- 所有代码可立即运行

### 逻辑完整性
- 边界条件处理完整
- 错误处理机制健全
- 资源释放/清理逻辑正确

## 2. 类型安全标准

### TypeScript/JavaScript
- 启用 strict 模式
- 避免使用 any 类型
- 优先使用接口定义数据结构
- 使用泛型提高复用性

### Python
- 使用 type hints
- 使用 dataclasses 定义数据结构
- 使用 Enum 替代魔法数字/字符串

## 3. 错误处理标准

### 异常处理
- 捕获具体异常，不使用 bare except
- 异常信息包含上下文
- 必要时记录日志
- 异常不泄露敏感信息

### 返回值设计
- 优先使用 Result/Either 模式
- 明确错误类型和成功状态
- 提供默认值的 safe fallback

## 4. 性能标准

### 资源管理
- 及时释放资源（文件、连接、内存）
- 使用连接池复用资源
- 避免内存泄漏

### 算法效率
- 选择合适的数据结构
- 避免不必要的循环嵌套
- 使用缓存减少重复计算

## 5. 安全标准

### 输入验证
- 验证所有外部输入
- 防止 SQL注入、XSS、CSRF
- 敏感数据加密存储

### 依赖管理
- 定期更新依赖
- 避免使用已知漏洞的库
- 最小化依赖数量
