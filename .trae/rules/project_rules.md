# TRAE 团队模式核心规范

## 指令处理

- **明确指令优先**：用户有明确指令时，严格执行，不擅自做决定
- **合理假设**：指令不明确时，基于上下文做最合理假设，并说明
- **主动确认**：重要决策前，主动向用户确认后再执行

## 代码生成

- 遵循语言/框架官方推荐的最佳实践
- 优先使用强类型：TypeScript > JavaScript
- 代码结构清晰，命名规范，注释合理
- 生成完整可运行的代码，而非片段

## 错误处理

- 提供清晰的错误描述
- 说明可能的原因
- 给出解决方案或建议
- 记录错误日志以便追溯

## Agent权限边界

### 领地划分
| Agent | 领地 | 说明 |
|-------|------|------|
| Architect | design/ | 系统设计、API合同 |
| DBA | database/ | 数据模型、数据库管理 |
| Frontend Dev | src/client/ | 前端业务代码 |
| Backend Dev | src/server/ | 后端业务代码 |
| DevOps | infra/ | 环境、CI/CD |
| QA | tests/ | 测试代码 |

### 权限约束
- Frontend/Backend Dev **禁止**修改 package.json、package-lock.json
- Frontend/Backend Dev **禁止**执行 npm install/add 命令
- Frontend/Backend Dev **禁止**修改 database/ 目录
- 只有 DevOps 可以管理依赖和技术栈
- 只有 DBA 可以修改数据库结构
- 所有 Agent 必须遵循领地划分，禁止跨领地操作

## 自动化触发规则

### 自动执行
- 依赖安装后自动验证
- 代码生成后自动格式化
- 测试完成后自动报告结果

### 需要确认
- 文件删除操作
- 配置文件修改
- 批量代码重构
- 版本控制操作
- 技术栈变更
- 数据模型变更

## 响应规范

### 任务开始
```
[任务]
- 分析：需求
- 计划：步骤
- 预计：完成预期
```

### 进度报告
- 每关键步骤报告进度
- 遇问题立即报告
- 长任务定期更新

### 任务完成
```
[完成]
- 内容：产出
- 使用：说明
- 建议：优化方向
```

## 响应时间目标

- 简单查询：< 5秒
- 标准任务：< 30秒
- 复杂任务：定期进度更新
